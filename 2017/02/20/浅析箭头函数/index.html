浅析 ES6 中的箭头函数

javascript ES6
前端
发布日期:   2017-02-20
文章字数:   1k
阅读时长:   3 分

定义：箭头函数是 ES6 中的表达函数的写法。即：用箭头定义函数 () => a + b

通常我们定义 JS 中的函数时是这么定义的

ES5：

function (a) {
	return a;
}

// 输出结果：a

这种写法我们已经习以为常了，但是总感觉这种写法太冗长了，那有没有一种方法用一行代码就能表达清楚的呢？答案当然是：有！

在 ES6 中新增加了一种定义函数的方法：箭头函数。写起来简单明了。如下：

ES6：

(a) => a;

// 输出结果：a

这样看着是不是就简洁的多了呢，不仅把 function 、 return 关键字省略了，连大括号 小括号都不用写了。有人问了：我都没写 return 关键字，它是怎么知道我要把 a 返回出来呢？那是因为箭头函数默认将箭头后面的代码 return 出来。（注意：这种机制仅限于单行代码）

那么问题来了，这是一个有参数的函数，而且只有一个参数。如果我要写没有参数或者有多个参数的函数该怎么写呢？

for example:

ES5：

 function (a, b) {
 	return a + b;
 }

 // 输出结果： a + b


 ES6：

 (a, b) => a + b

 // 输出结果： a + b

只需要在箭头前面加一个括号，在括号里面写上要传入的参数并用逗号将参数分割就行。依然很简洁！

当然了，这只是最简洁的函数，箭头函数有两种表达格式，一种就是上面的，只包含一个表达式，如果表达式比较复杂怎么办呢？这时就不能再省略 {} 和 return 啦。

ES5：

function (a) {
	if (a > 0) {
		return a;
	}else {
		return - a;
	}
}

// 输出结果： a or - a

ES6：

a => {
	if (a > 0) {
		return a;
	}else {
		return - a;
	}
}

// 输出结果： a or - a

这是单个参数的写法，如果是多个参数或者无参数，同样在 => 前面加上 (option1, option2) 就行啦。

箭头函数中有一种特殊情况： 如果要返回一个对象，那么就要注意，这么写的会报错

(a) => {
  name: TheInlaidHarp;
};

// SyntaxError

因为这样写和函数体的 {…} 有语法冲突，所以要改为

(a) => ({ name: TheInlaidHarp });

// OK

到这个地方整体看起来箭头函数与之前的函数也没有太大的区别，别急！下面就介绍一下箭头函数与普通的函数最大区别，也就是箭头函数的特性：

箭头函数对 this 的处理方法与普通的函数是不一样的，我们通过一个例子来理解

var object = {
  number: 1,

  fn: function () {
    console.log(this.number);
  },

  test: function () {
    (function () {
      this.fn();
    })();
  },
};

object.test();

// this.fn is not a function

这样写会报错的原因就是：test 函数里面的函数调用的时候 this 是指向全局的。那么怎么才能让这个 this 指向 object 呢？我们来修改一下代码

var object = {
  number: 1,

  fn: function () {
    console.log(this.number);
  },

  test: function () {
    var _this = this;

    (function () {
      _this.fn();
    })();
  },
};

object.test();

// 1

我们在外部定义一个变量 _this 来保存 this 就 OK 啦！

我们刚才说过：在箭头函数里面对 this 的处理方法是不一样的，箭头函数的 this 始终指向函数定义的 this ，而非函数调用时的 this 。所以我们将上面的代码用箭头函数表达

var object = {
  number: 1,

  fn: function () {
    console.log(this.number);
  },

  test: function () {
    (() => this.fn())();
  },
};

object.test();

// 1

所以这次的函数执行结果的 this 指向依然是 object;

但是我们要注意的是：箭头函数里的 this 指向是不会改变指向对象的。我们知道 javascript 中的 call 和 apply 方法可以改变 this 指向，但是这两个方法在箭头函数中是无效的。

var number = 1;
var object = {
  x: 10,

  test: () => this,
};

console.log(object.test()); // window

console.log(object.test.call(object)); // 依然是 window
